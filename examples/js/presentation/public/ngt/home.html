What is Isotope?

What does it do and why is it necessary?

Speech Recognition

Speech recognition hardware is of particular interest at the Universy of Stellenbosch. A critical aspect of these devices is the ability to convey input to the host machine, requiring the implementation of a communications protocol.

Universal Compatibility

To simplify the development of these devices it makes sense to select a protocol which has already been widely adopted and implemented. The USB Human Interface Device protocol meets these requirements perfectly - with drivers available for most major operating systems.

Simple Implementation

Rather than develop a device specific implementation, it was decided that a simple addon board which could be ported to a number of different master devices would offer the best capacity for future expansion as embedded device performance increases.


Device Architecture

What components make up Isotope?

Master Device

The master device, in this case a Raspberry Pi computer, is responsible for determining when to execute each emulation operation. This can be driven by a web frontend as we will see here, a voice recognition engine or a simple cron job.

Isotope

The emulation device accepts an emulation command from the master device and converts it into a form which is understandable by the host device - using the USB HID specification as a basis for this.

Host Device

The host device implements a USB HID host mode driver and will automatically interpret the emulation commands it receives as though they were true user input. This makes Isotope almost completely indistinguishable from a standard mouse and keyboard.
